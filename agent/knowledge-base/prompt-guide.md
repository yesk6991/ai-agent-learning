# Prompt 工程指南

## 一、基础原则

### 1. 清晰明确
❌ "帮我写个功能"
✅ "用 JavaScript 写一个函数，接受一个数字数组，返回其中所有偶数的平方和，要求有 JSDoc 注释和类型标注"

### 2. 给出上下文
❌ "这段代码有什么问题？"
✅ "我正在写一个 React 组件，这段 useEffect 导致了无限渲染，请帮我分析原因"

### 3. 指定输出格式
❌ "列举几种排序算法"
✅ "列举 5 种排序算法，用以下 JSON 格式输出：
[{name: '算法名', timeComplexity: '时间复杂度', stable: true/false}]"

## 二、高级技巧

### Chain-of-Thought（思维链）
在问题末尾加上"请一步步思考"或"Let's think step by step"，
可以让模型展示推理过程，显著提升逻辑推理题的正确率。

**原理**：模型在生成推理步骤时，实际上是在进行"慢思考"，
每一步的中间结论都为下一步提供了更好的上下文。

### Few-shot（少样本学习）
给出 2-3 个输入-输出示例，让模型理解你期望的格式和风格。

**示例**：
```
将以下句子转换为正式语气：
输入：这东西不行  输出：该产品存在质量问题
输入：太贵了      输出：价格偏高，性价比有待提升
输入：还行吧
```

### ReAct（Reasoning + Acting）
让模型交替进行推理和行动，这是 Agent 的核心范式：

```
问题：北京今天适合户外运动吗？

思考1：我需要知道北京今天的天气 → 行动1：调用天气查询工具
观察1：北京 晴天 28°C 空气质量优
思考2：28°C 晴天，空气质量好，适合户外运动 → 行动2：给出建议
```

## 三、System Prompt 设计

System Prompt 定义了 AI 的"人格"和行为边界，好的 System Prompt 应包含：

1. **角色定义**：你是一个XX专家
2. **能力边界**：你擅长什么，不擅长什么
3. **行为规则**：回答风格、语言偏好、安全约束
4. **输出格式**：默认的输出格式要求
5. **边界情况**：遇到不确定的问题怎么办

## 四、Prompt 模板化

将 Prompt 抽象为模板，支持变量替换，便于：
- A/B 测试不同 Prompt 版本
- 批量处理相似任务
- 团队协作和版本管理

模板语法示例：
```
你是一个{{domain}}领域的专家。
请用{{tone}}的语气回答以下问题：
{{question}}

要求：
- 回答不超过{{max_words}}字
- 必须包含至少{{min_examples}}个示例
```
