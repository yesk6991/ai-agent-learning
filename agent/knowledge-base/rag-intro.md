# RAG 入门指南

## 一、为什么需要 RAG

LLM 有两个根本性的局限：
1. **知识截止**：训练数据有时间限制，不知道训练之后发生的事
2. **幻觉问题**：不确定时会"编造"看起来合理的答案

RAG 通过"先检索，再生成"的方式解决这两个问题。

## 二、RAG 的核心流程

```
                    索引阶段（离线）
                    ┌──────────────┐
  文档 ──→ 分块 ──→ 向量化 ──→ 存储索引
                                  │
                    查询阶段（在线）│
                    ┌──────────────┘
                    │
  用户问题 ──→ 问题向量化 ──→ 相似度检索 ──→ 取出Top-K片段
                                                    │
                                                    ▼
                              问题 + 检索片段 ──→ LLM 生成回答
```

## 三、文档分块（Chunking）

### 分块策略
1. **固定大小分块**：每块 N 个 Token，相邻块有 M 个 Token 重叠
   - 优点：简单，Token 数可控
   - 缺点：可能在句子中间断开

2. **语义分块**：按段落、章节、标题等自然边界分块
   - 优点：保留语义完整性
   - 缺点：块大小不均匀

3. **递归分块**：先按大单位（章节）切，超过限制再按小单位（段落）切
   - 优点：兼顾语义和大小
   - 缺点：实现较复杂

### 重叠（Overlap）
相邻分块之间保留一部分重叠内容，避免关键信息恰好在切分点被截断。
通常 overlap = chunk_size * 10%~20%

## 四、向量化（Embedding）

将文本转换为固定维度的数值向量，使语义相近的文本在向量空间中距离也近。

### 常用 Embedding 模型
- OpenAI: text-embedding-3-small/large
- 开源: BGE、M3E、GTE
- 本地方案: TF-IDF（本项目使用）

### TF-IDF 简化方案
不依赖外部 API，用词频-逆文档频率计算文本向量：
- TF（词频）：某个词在文档中出现的频率
- IDF（逆文档频率）：越稀有的词越重要
- TF-IDF = TF × IDF

## 五、相似度检索

### 余弦相似度
最常用的向量相似度计算方法，衡量两个向量方向的相似性：
cos(A, B) = (A·B) / (|A| × |B|)

值域 [-1, 1]，越接近 1 表示越相似。

### Top-K 检索
取出与查询向量最相似的 K 个文档片段，通常 K = 3~5。

## 六、上下文注入

将检索到的片段作为上下文拼入 Prompt：

```
请根据以下参考资料回答问题。如果资料中没有相关内容，请说明。

参考资料：
---
[1] 片段1内容...
[2] 片段2内容...
[3] 片段3内容...
---

问题：用户的提问
```

## 七、RAG 的进阶优化

1. **查询重写**：将用户问题改写为更适合检索的形式
2. **混合检索**：结合关键词检索和语义检索
3. **重排序**：先用粗检索取 Top-20，再用精细模型重排取 Top-5
4. **多路召回**：同时从多个角度检索，合并结果
