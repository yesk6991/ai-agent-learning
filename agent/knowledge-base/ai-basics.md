# AI 基础概念

## 什么是大语言模型 (LLM)

大语言模型（Large Language Model）是一种基于 Transformer 架构的深度学习模型，通过在海量文本数据上进行自监督学习，获得了理解和生成自然语言的能力。

### 核心原理
- **自回归生成**：逐个 Token 预测下一个最可能的 Token，直到生成结束符
- **注意力机制**：通过 Self-Attention 计算文本中每个位置对其他位置的关注权重
- **上下文窗口**：模型能"看到"的最大 Token 数量，如 128K、200K

### 主流模型
- **Claude 系列**（Anthropic）：Haiku（快速）、Sonnet（均衡）、Opus（深度推理）
- **GPT 系列**（OpenAI）：GPT-4o、o1、o3
- **开源模型**：Llama、Qwen、DeepSeek

## 什么是 Agent

Agent（智能体）是一种能够自主感知环境、做出决策并执行动作的 AI 系统。与传统的"单轮问答"不同，Agent 具备：

1. **感知**：理解用户意图和当前环境状态
2. **推理**：通过思维链（CoT）进行多步推理
3. **行动**：调用工具、检索知识库、与外部系统交互
4. **反思**：评估自己的输出，必要时修正

### Agent 的典型架构

```
用户输入 → Prompt 模板 → LLM 推理 → 工具调用 → 结果回传 → LLM 生成 → 输出
              ↑                                                    |
              └──────────── 知识库检索（RAG）───────────────────────┘
```

## 什么是 RAG（检索增强生成）

RAG（Retrieval-Augmented Generation）是一种让 LLM 利用外部知识的技术：

1. **索引阶段**：将文档分块 → 向量化 → 存入向量数据库
2. **检索阶段**：用户提问 → 问题向量化 → 相似度检索 → 取出相关文档片段
3. **生成阶段**：将检索到的片段作为上下文，让 LLM 生成回答

RAG 解决了 LLM 的两大问题：
- **知识过时**：训练数据有截止日期，RAG 可以实时获取最新信息
- **幻觉问题**：LLM 可能"编造"答案，RAG 提供了事实依据

## 什么是 Prompt Engineering

Prompt Engineering（提示词工程）是设计和优化输入提示词以获得更好模型输出的技术。

### 核心技巧
1. **Zero-shot**：不给示例，直接提问
2. **Few-shot**：给出几个示例，引导输出格式
3. **CoT（Chain-of-Thought）**：要求模型"一步步思考"
4. **ReAct**：交替进行推理和行动
5. **结构化输出**：用 JSON Schema 约束输出格式
6. **角色设定**：通过 System Prompt 定义 AI 的行为方式
